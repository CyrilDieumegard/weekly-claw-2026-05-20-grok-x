# Weekly Claw - Grok x OpenClaw - 20 mai 2026

## Résumé court

La session a démarré avec le rituel des nouveaux : 30 secondes pour se présenter, expliquer son usage de l’IA et son blocage du moment. Deux profils ressortent fortement : Mokhles, côté finance/n8n/automation, et David, ancien IT manager Big Pharma qui lance une activité d’audit IT/process/résilience pour PME et médecins avec une contrainte forte de données en Suisse ou en local.

Le fil rouge annoncé était l’intégration Grok/xAI dans OpenClaw via OAuth X. La discussion a vite élargi le sujet : comment donner à un agent local une source d’information fraîche, comment éviter les automatisations de contenu dangereuses, et comment garder le contrôle sur les données.

## Points clés

- Grok/xAI dans OpenClaw : connexion OAuth avec X, accès à des signaux temps réel, potentiel pull/push autour des posts X.
- n8n local : OpenClaw peut installer et piloter n8n sur une machine locale pour créer des workflows sans sortir les données.
- Santé / audit / Suisse : pour David, le combo OpenClaw + n8n + modèle local répond bien aux contraintes médecins, confidentialité et souveraineté.
- Apify : présenté comme une brique très utile pour scraper LinkedIn, TikTok, Instagram, jobs, concurrents ou prospects via API key.
- Warning important : éviter le posting automatique non validé humainement. Risque de ban et contenu médiocre.
- Souveraineté : débat sur Infomaniak, Proton, Open Data Suisse, RAG, data rooms locales et machines isolées.
- LocalClaw : démo de l’interface pour installer, mettre à jour, gérer modèles cloud/auth/local, projets, agents et cron jobs.
- Hermès vs OpenClaw : Hermès paraît plus strict, mais moins convaincant pour le local et probablement plus gourmand en contexte.
- Hardware local : Mac mini, Mac Studio, NVIDIA DGX et machines dédiées deviennent des sujets sérieux pour faire tourner des agents localement.
